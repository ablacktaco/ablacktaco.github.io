Array - Polynomial

國立交通大學開放式課程

Array 是一段連續的記憶體空間，有大小限制。
可視為一種 Dictionary，有原生不可變更的 key，為zero base、公差為1的等差級數(也就是index)。

Array 可以用來存放 polynomial，在與指數項相對應的 index 空間內存放該指數項的係數。例如 x⁴+10x³+3x²+1 可以寄記做 [1, 0, 3, 10, 1]。
相加 2 多項式 a、b 時，需比較 a、b 的長度。若 a>b ，則將 a 末項的值插入 result 的首項後從 a 裡移除，反之 b 亦然。而當 a=b 時，則將 a、b 的末項值相加後插入 result 的首項，並分別將 a、b 的末項刪除。

但這個做法如果遇到中間許多項次係數為 0 的狀況 — 例如 x¹⁰⁰⁰+1 ，則會造成許多空間的浪費，於是衍生了第 2 種做法 — 僅記錄係數不為 0 的項次以及該項次的係數。例如 2x¹⁰⁰⁰+1 可以記做 [{coef: 2, exp: 1000}, {coef: 1, exp:0}]。

沒有了指數項與 index 相依的問題，便可以把多個 polynomial 記在同一個 array 裡。而用以區分各多項式，須額外記錄各多項式的 start 以及 finish 的 index 。

在這種存放方式下多項式 a、b 相加時，則需比較 a.start 與 b.start 的 exp (指數項) ，若 [a.start].exp > [b.start].exp，則將 a.start 的 coef 及 exp 複製到 array 有序的空位上(free)，複製後 free 的 index 需要後移以接收下一筆資料。接下來比較 a.start + 1 與 b 的 exp (b 的 exp 較大或相等的處理方式與前一個方法相同)，直到 a.finish 及 b.finish 都被複製為止。

但一開始也提到過，array 是一個有大小限制的記憶體空間。在 free 後移到array 的盡頭時，便沒有空間再存放資料。這時就必須回收(destructor)已做過運算且不會再使用的 polynomial，回頭利用被回收的記憶體空間。

這時又衍生了一個問題 — release 後的記憶體空間不一定是連續的。但由於多項式的區分是利用紀錄多項式的 start 以及 finish，不連續的存放空間就會造成資料的誤判，所以在回收多項式的同時必須伴隨著資料的搬移，以確保 release 後的空間是連續的。
