國立交通大學開放式課程

Array 也可以用來存放矩陣，通常用二維陣列存放。

但如果遇到 sparse matrix (稀疏矩陣，矩陣中大部分的 element 為 0)，就會跟大部分係數為 0 的多項式產生一樣的問題，也就是空間的浪費。於是同樣有了只記錄 element 不為 0 的位置及 value 的做法，例如[[3, 0, 0], [0, 0, 2], [0, 5, 0]] 可以改寫成 [{row: 0, column: 0, value: 3}, {row: 1, column: 2, value: 2}, {row: 2, column: 1, value: 5}]，紀錄方式為 row major(先記錄完第 0 列所有的 element 再記錄第 1 列)。

而矩陣轉置(transpose)時，僅需交換 row 與 column 的 value 即可…但事情總是沒有想像的這麼簡單。例如上述的例子，互換後為 [{row: 0, column: 0, value: 3}, {row: 2, column: 1, value: 2}, {row: 1, column: 2, value: 5}]，意義上是轉置了，卻沒有符合 row major。為了符合 row major 就必須重新排列 array ，耗費的時間成本就會大幅提升。
(範例程式取自課程投影片)

跑了 2 個 for 迴圈，複雜度為 O(columnterm)，term 在 worst case 下為 rowcolumn。

而比較聰明的做法是先進行一次查找，統計各個 row 裡有幾個 element，計算出各個 row 在 array 裡起始的 index 後再進行轉置。這麼一來轉置時只需要判斷 row 的值就知道要存放在哪個位置，可以省去重新排列的時間，複雜度僅有 O(row*column)。
