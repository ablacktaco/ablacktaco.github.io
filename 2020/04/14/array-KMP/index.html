KMP algorithm

國立交通大學開放式課程

Knuth-Morris-Pratt(KMP) 是字串查找的演算法。

一般查找字串比較直覺的做法是：將字串的第一個字(T1)與文章內容(Pn)一個一個做比對，比對失敗就 shift 一格(比對 T1/Pn+1)；成功則繼續比較(T2/Pn+1)，直到想查找的字串都被比對成功為止。

但這個方法存在明顯時間上的浪費：例如我們要找 pikachu，但比對到 piano 的時候在 T3(k)/P3(a) 才發生錯誤。
這時我們要重新比較 T1/P2 有沒有相同 — 看也知道沒有，因為從剛剛的比對已經知道 T1(p)=P1(p)、T2(i)=P2(i)，但 T1 又不等於 T2，同理也不可能等於 P2。

如果可以的話，我們不希望再浪費時間比對已經知道不可能成功的字元。但說是這樣說，要如何告訴電腦這件事情？

KMP 就是以這個思維做出發點的字串查找演算法，他利用想查找字串的 failure function，來告訴電腦如果比對失敗的話，可以一次跳過幾個字元來節省比對的時間。

Failure Function

failure function 是字串的潛在匹配個數。也就是說如果發生比對失敗，當下比對字元前的字元組與字首的相似程度。

如上圖所示，字串在 T(v)/P(x) 時發生比對失敗，但 x 前的字元組 z 與字首的字元組 z 相同，所以字首就可以直接 shift 到已知比對成功的字元組，節省重複比對的時間。
而 failure array 則紀錄了每個字元的重複字元組 z 的長度。

舉例來說，字串 ababaca。
T1 因為是字首，shift 到自己只會進入無限 loop，所以 failure function 為 0；T2 與字首的相同字元數為 0，failure function 也是 0。

T23(ba) 與字首的相同字元組只有 T3/T1，failure function 為 1。

T234(bab) 與字首的相同字元組有 T34/T12，failure function 為 2。

觀念是清楚了，但總不可能每次都用工人智慧推導，該如何用程式讓計算機自己找到 failure array 呢？

我們可以從上面的範例找出規律：
index=0 時，failure function=0；
index=1 時，比對 T1/T0；
index=2 時，比對 T12/T01，如果失敗則比對 T2/T0；
index=3 時，比對 T123/T012，失敗則比對 T23/T01，再失敗則比對 T3/T0，以此規律遞減。

每個字元的最高比對次數，都剛好與該字元的 index(i) 相同，且比對的字元組剛好為後 i 個字元/前 i 的字元、後 i-1 個字元/前 i-1 的字元、後 i-2 個字元/前 i-2 的字元…
